Pandoc

A command-line universal document converter — the gold standard for batch and pipeline use, converting between 30+ formats. PDF output requires a LaTeX install (~1.5GB via TeX Live on macOS), and the default engine fails on CJK: you must pass --pdf-engine=xelatex -V mainfont="Noto Sans CJK SC" with the fonts installed. Best for: engineers scripting batch conversion, academic publishing, CLI-comfortable writers.

Dillinger

A browser Markdown editor with live preview; the hosted version is ad-supported and can sync to Dropbox/Drive/GitHub. PDF export inherits system fonts, so CJK that looks fine in preview can still come out as tofu. Best for: one-off edits and exports of Latin-script documents.

StackEdit

Browser editor with cloud sync and MathJax math. PDF goes through the browser print pipeline with limited styling control; fonts depend on the system. Best for: writers who need cloud sync and math support.

Markdown PDF (VS Code extension)

Exports the current VS Code file to PDF/HTML/PNG/JPEG; first run downloads Chromium (~170MB). Uses Chromium's font stack — fine when CJK fonts are installed, configurable via CSS @font-face. Best for: developers already living in VS Code.

md-to-pdf (npm)

A Node.js CLI/library that renders Markdown to PDF through Puppeteer — built for CI/CD pipelines, themeable via CSS. Needs Node + Chromium. Best for: engineering teams generating PDFs from docs in CI.

Typora

A polished desktop WYSIWYG editor (one-time purchase). Beautiful output, no ads or telemetry, desktop only. CJK handling depends on the theme's font stack. Best for: solo writers who want offline polish and don't mind paying.

Online2PDF

A generic web file converter that handles Markdown incidentally. Files are uploaded to a server (privacy risk for confidential docs), rendering of tables/task lists is inconsistent. Best for: mixed-format batch conversion where Markdown is a side case.

Pandoc

命令行通用文档转换器，可在 30+ 种格式间互转，脚本批处理的事实标准。PDF 输出需安装 LaTeX（约 1.5GB），且默认引擎不支持中文——必须 --pdf-engine=xelatex -V mainfont="Noto Sans CJK SC" 并安装对应字体。最适合：用脚本批量转换的工程师、学术出版。

Dillinger

带实时预览的浏览器编辑器，托管版有广告，可连 Dropbox/Google Drive/GitHub。PDF 使用系统字体，预览正常、导出变豆腐的情况并不少见。最适合：纯英文文档的临时编辑。

StackEdit

支持云盘同步和 MathJax 数学公式的浏览器编辑器。PDF 走浏览器打印通道，样式自由度低，中文依赖系统字体。最适合：需要云同步与数学公式的写作者。

Markdown PDF（VS Code 扩展）

把当前 Markdown 文件导出为 PDF/HTML/PNG/JPEG，首次使用下载 Chromium（约 170MB）。系统有中文字体即可正常显示，可用 CSS 嵌字体。最适合：本来就在 VS Code 里工作的开发者。

md-to-pdf（npm）

用 Puppeteer 把 Markdown 转 PDF 的 Node.js CLI，专为构建流水线设计，可在 CSS 中引入 Noto 字体稳定渲染中文。最适合：CI/CD 中从文档生成 PDF 的工程团队。

Typora

桌面 WYSIWYG 编辑器，一次性买断。输出精致、无广告无遥测，仅桌面。中文表现取决于主题字体栈。最适合：独立写作、不介意付费的人。

Online2PDF

通用网页转换器，Markdown 只是顺带支持。文件上传到服务器（机密文档有风险），表格/任务列表渲染不稳定。最适合：混合格式批量转换的一次性场景。
