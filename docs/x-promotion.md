## Week 1 — Warm-up (no promotion)

### Day 1 — Markdown tip
```
markdown tip that took me embarrassingly long to learn:

you don't need ### for everything. bold works fine for
lightweight structure and the doc looks way less cluttered

**What:** the thing
**Why:** the reason
**How:** the step

that's it. no header hierarchy needed
```

---

### Day 2 — Obsidian tip
```
obsidian callouts are genuinely underrated

> [!WARNING]
> don't forget this

> [!TIP]
> faster way to do it

been using them for 6 months and still finding new spots
where they make notes 10x more readable. also they export
cleanly which is a nice bonus
```

---

### Day 3 — Markdown tip
```
unpopular opinion: markdown tables aren't that hard

| thing | status |
|-------|--------|
| done  | ✅     |
| nope  | ❌     |

i avoided them for years because they looked weird in raw text.
now i use them constantly. github renders them, obsidian renders them,
basically everywhere renders them
```

---

### Day 4 — Obsidian tip
```
my meeting note template, stolen from too many late nights:

## [meeting name] — [date]
**who:**
**goal:** one sentence

### what we decided
-

### who does what
- [ ] task — person — by when

took me way too long to have a consistent format.
now i just duplicate this every time
```

---

### Day 5 — Markdown tip
```
reminder that markdown task lists exist

- [x] done
- [ ] not done yet
- [ ] also not done

works in github, obsidian, notion, bear, basically anything.
way faster than writing "TODO:" everywhere and then ctrl+f-ing
for it later
```

---

### Day 6 — Obsidian tip
```
started using frontmatter in obsidian and honestly can't go back

---
status: draft
tags: [work, client]
date: 2026-02-27
---

sounds overkill until you have 400 notes and need to find
"all drafts from last month". then suddenly it's very worth it
```

---

### Day 7 — Markdown tip (bridges into tool)
```
the thing i actually like about markdown:

opened a note i wrote in 2019 today. just worked.
no "this file format is no longer supported", no weird
encoding, no missing fonts. just text.

kind of rare for a file format to age that well
```

## Week 2 — Promotion Posts

### Day 8 — Tool intro (pin this)
```
built a thing: markdown.free

converts .md files to PDF, Word, or HTML. no account,
no watermarks, files aren't stored anywhere.

made it because every other converter i tried either
wanted me to sign up or slapped their logo on my output.
neither felt right for a utility tool

https://www.markdown.free
```

---

### Day 9 — Obsidian angle
```
if you use obsidian and occasionally need to send notes
to someone who doesn't:

export as .md → drop on markdown.free → send the PDF

tables, code blocks, callouts all render properly.
takes about 15 seconds total and the output looks clean

no one needs to install obsidian to read it
```

---

### Day 11 — AI output angle
```
chatgpt and claude write in markdown

which means if you want to actually save something useful
they generated — not just copy-paste into a doc —
you can paste the output into markdown.free and get a
proper PDF or Word file out

small thing but saves a bunch of manual reformatting
```

---

### Day 13 — README angle
```
random workflow that's been useful:

before sending a README to a non-technical stakeholder,
convert it to PDF first. they don't want to look at
raw markdown or click through to github

markdown.free does this in one drag and drop.
the rendered output is clean enough to actually share
```

---

### Day 14 — Privacy / reply bait
```
genuine question: does anyone else feel weird about
uploading documents to random converters?

like you paste your work notes or client brief into
some site and have no idea what happens to it after

that's mostly why i built markdown.free the way i did —
files are gone immediately after conversion, nothing stored.
felt like the obvious way to do it but apparently it's not common
```

- Don't follow-unfollow — just engage genuinely in the communities
